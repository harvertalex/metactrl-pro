## 🔧 Быстрые команды

```bash
# Windows: Регенерирование B64 + Деплой
deploy.bat full

# Bash: Регенерирование B64 + Деплой
./makefile.sh deploy-full

# Просмотр логов Apache
./makefile.sh logs

# Проверка сервера
./makefile.sh check

# SSH подключение
ssh -i ~/.ssh/tessa-bot root@192.248.190.182
```

## 📋 Читай в зависимости от задачи

### 🎯 Я хочу добавить новое правило
1. Прочитать: **CLAUDE.md** → раздел "Правила генератора"
2. Отредактировать: **bookmarklet.js** (~строки 1100–1586)
3. Регенерировать B64 и деплой: `deploy.bat full` или `./makefile.sh deploy-full`

### 🎯 Я хочу изменить UI
1. Прочитать: **CLAUDE.md** → "Примечания по кодированию"
2. Отредактировать: **bookmarklet.js** (~строки 400–1100, функция `mountUI()`)
3. Регенерировать B64 и деплой: `deploy.bat full`

### 🎯 Я хочу обновить сервер вручную
1. Прочитать: **PROJECT-STRUCTURE.md** → раздел "Сервер"
2. SSH команды для ручного управления Apache, логов, перезагрузки

### 🎯 Я новый на проекте (первая сессия)
1. Прочитать: **CONTEXT-CHECKLIST.md** → раздел "Когда начинаешь работать с проектом"
2. Прочитать: **CLAUDE.md** → первые 3 раздела
3. Запустить: `./makefile.sh check` (убедиться что всё работает)

### 🎯 Деплой не работает / что-то сломалось
1. Прочитать: **CONTEXT-CHECKLIST.md** → раздел "Трабушутинг"
2. Запустить: `./makefile.sh status` или `ssh -i ~/.ssh/tessa-bot root@192.248.190.182 "systemctl status apache2"`

### 🎯 Я хочу понять архитектуру
1. Прочитать: **PROJECT-STRUCTURE.md** полностью
2. Прочитать: **CLAUDE.md** → "FB API Ограничения", "Rate Limiting"

## ✨ Ключевые факты

1. **B64 регенерация** — обязательна после каждого изменения `bookmarklet.js`
2. **API ограничения** — некоторые поля работают только на CAMPAIGN level
3. **Rate limiting** — FB API может отклонить если слишком много правил сразу
4. **Спенд пороги** — спенд должен быть в 1.5–2x от порога перед запуском правила
